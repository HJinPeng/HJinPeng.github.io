移动开发——viewport

1. 什么是viewport？

  • viewport是设备屏幕上用来显示网页的那块区域
  • viewport可能比浏览器可视区域大，也可能小，默认情况下是大。
  • 移动设备的浏览器都会把自己默认的viewport设置为980px或1024px

2. PPK（大神）的关于三个viewport的理论

  1. layout viewport

  2. visual viewport

  3. ideal viewport——理想viewport

  • ideal viewport并没有一个固定的尺寸，不同的设备拥有有不同的ideal viewport。

  • 所有的iphone的ideal viewport宽度都是320px，无论它的屏幕宽度是320还是640，也就是说，在iphone中，css中的320px就代表iphone屏幕的宽度。

  • 安卓设备就比较复杂了，有320px的，有360px的，有384px的等等，关于不同的设备ideal viewport的宽度都为多少，可以到http://viewportsizes.com去查看一下，里面收集了众多设备的理想宽度。

  • ideal viewport 的意义在于，无论在何种分辨率的屏幕下，那些针对ideal viewport 而设计的网站，不需要用户手动缩放，也不需要出现横向滚动条，都可以完美的呈现给用户。

3. 利用meta标签对viewport进行控制

  • 移动设备默认的viewport是layout viewport
  • 进行移动设备网站的开发时，我们需要的是ideal viewport
  • 把下面这个东西复制到我们的head标签中：
    <meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=0">
    该meta标签的作用是让当前viewport的宽度等于设备的宽度，同时不允许用户手动缩放。也许允不允许用户缩放不同的网站有不同的要求，但让viewport的宽度等于设备的宽度，这个应该是大家都想要的效果，如果你不这样的设定的话，那就会使用那个比屏幕宽的默认viewport，也就是说会出现横向滚动条。
    其中initial-scale=1 解决了 iphone、ipad的毛病，width=device-width则解决了IE的毛病。
