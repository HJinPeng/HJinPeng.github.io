Bootstrap

一、 概览

1. HTML5 文档类型

2. 移动设备优先

3. 布局容器

.container 类用于固定宽度并支持响应式布局的容器。

.container-fluid 类用于 100% 宽度，占据全部视口（viewport）的容器。

由于 padding 等属性的原因，这两种 容器类不能互相嵌套。

二、 栅格系统

1. 栅格系统

Bootstrap 提供了一套响应式、移动设备优先的流式栅格系统，随着屏幕或视口（viewport）尺寸的增加，系统会自动分为最多12列。

2. 媒体查询

/ 超小屏幕（手机，小于 768px） /
/ 没有任何媒体查询相关的代码，因为这在 Bootstrap 中是默认的（还记得 Bootstrap 是移动设备优先的吗？） /

/ 小屏幕（平板，大于等于 768px） /
@media (min-width: @screen-sm-min) { … }

/ 中等屏幕（桌面显示器，大于等于 992px） /
@media (min-width: @screen-md-min) { … }

/ 大屏幕（大桌面显示器，大于等于 1200px） /
@media (min-width: @screen-lg-min) { … }

3. 栅格参数

4. 响应式列重置

在某些阈值时，某些列可能会出现比别的列高的情况。为了克服这一问题，建议联合使用 .clearfix 和 响应式工具类。

5. 列偏移

使用 .col-md-offset- 类可以将列向右侧偏移。这些类实际是通过使用 选择器为当前元素增加了左侧的边距（margin）。例如，.col-md-offset-4 类将 .col-md-4 元素向右侧偏移了4个列（column）的宽度。

6. 嵌套列

7. 列排序

通过使用 .col-md-push- 和 .col-md-pull- 类就可以很容易的改变列（column）的顺序。

三、 排版

1. 标题

HTML 中的所有标题标签，h1 到 h6 均可使用。另外，还提供了 .h1 到 .h6 类，为的是给内联（inline）属性的文本赋予标题的样式。
在标题内还可以包含 < small > 标签或赋予 .small 类的元素，可以用来标记副标题。

2. 页面主体

  • Bootstrap 将全局 font-size 设置为 14px，line-height 设置为 1.428。这些属性直接赋予 < body > 元素和所有段落元素。另外，< p > （段落）元素还被设置了等于 1/2 行高（即 10px）的底部外边距（margin）。
  • 通过添加 .lead 类可以让段落突出显示。<p class="lead">...</p>

物理像素 和 CSS像素

发表于 2018-05-05 | 分类于 移动端

移动开发——物理像素/css像素

1. 物理像素

理解：每个设备的物理像素是固定的。不同设备的物理像素可能不同，看分辨率。

2. CSS像素

理解：

  • CSS样式代码中使用的是逻辑像素。
  • 在一个设备中，物理像素与逻辑像素相等，将不会产生任何问题。
  • 在iphone 4中，物理像素是640px×960px，而逻辑像素数为320×480px。因此，需要使用大约4个物理像素来显示一个CSS像素。
